1 Overview

The MU Module Catalogue is a fully public, browser-based tool for exploring Maynooth University's complete module offering. It requires no login, no server connection at runtime, and works offline once loaded. It is part of the MU Finder suite alongside the MU Programme Finder.

The catalogue currently covers 2,866 modules across 48 departments and three faculties. Module content descriptions, learning outcomes, ECTS credits, assessment components, and teaching & learning data are drawn directly from the MU public course catalogue for Academic Year 2026.

Primary audiences: current and prospective students exploring MU's curriculum; academic staff reviewing module offerings across departments; university administrators and external stakeholders.

Key capabilities

  • Full-text search across module codes, names, content descriptions, and learning outcomes
  • Filter by faculty, department, level (Undergraduate / Postgraduate), and skill tags
  • Save modules to a personal shortlist (persisted locally, no login required)
  • View a summary of saved modules including Data/Digital skill coverage and departmental spread
  • Expand any module card to read the full content description, numbered learning outcomes, and Critical Skills mappings where available
  • Open a Full Details modal for any module showing ECTS credits, semester, international availability, teaching & learning hours breakdown, assessment components, supplemental/resit information, and a link to the official MU timetable
  • International modules are identified with a sky-blue badge — these are modules available to short-term, visiting international students
  • Accessible font size toggle (three sizes, persisted across sessions)

2 Data & Digital Tagging

Each module in the catalogue may carry one or both of two skill tags, representing the university's audit of data and digital literacy content embedded across the full curriculum.

Data

The module has been identified as containing meaningful data skills content — including data literacy, statistical reasoning, data analysis, quantitative methods, or data management. Tagged by department heads or delegated staff following review of the module content description and learning outcomes.

Digital

The module has been identified as containing meaningful digital skills content — including computing, software use, digital tools, information systems, coding, or digital communication. Tagged by department heads or delegated staff.

Background

The tagging was conducted as a university-wide initiative to estimate the extent of data and digital skills embedded in MU's full module catalogue. An initial automated pass using AI (Gemini) analysed module learning outcomes and content descriptions to pre-tag modules. This was followed by a human review pass — each department's Head of Department (or delegated staff member) confirmed, corrected, or extended the tags using a bespoke module management application built for this purpose.

Near-complete human tagging was achieved across the full module catalogue — described internally as a significant institutional first.

Note: Tags reflect the state of the catalogue at the time of export. A module without a tag has not been positively identified as containing data or digital content — it does not necessarily mean such content is absent.

3 Browsing & Filtering

The catalogue defaults to showing all Undergraduate modules on load, sorted alphabetically by department then module code. Four filter controls are available at the top of the catalogue:

FilterOptionsBehaviour
Faculty All Faculties · FAH · FSE · FSS (+ CIF, VPSL) Restricts the department dropdown to departments within the selected faculty
Department All Departments · [list] Cascades from the faculty filter. Shows all 48 departments when no faculty is selected
Level All Modules · Undergraduate · Postgraduate · Unknown Defaults to Undergraduate. Select All Modules to see every level
Data / Digital Toggle buttons Show only modules tagged with the selected skill. Both can be active simultaneously via the search; only one toggle is active at a time

All filters combine — for example, selecting Faculty of Science & Engineering + Postgraduate + Digital returns only FSE postgraduate modules tagged as containing digital skills content.

A Clear all link resets every filter and returns to the default Undergraduate view.

Pagination

Results are shown 48 at a time. A Show more button at the bottom loads the next 48. The result count displayed above the grid always reflects the total number of matching modules, not just the visible ones.

4 Search

The search box performs full-text fuzzy search across the currently filtered pool of modules using Fuse.js. Searching begins once at least 2 characters have been entered.

Fields searched

FieldWeightNotes
Module codeHigh (3×)e.g. CS101, MA201
Module nameMedium (2×)Full module title
Department nameLow (1×)e.g. Computer Science
Module contentLower (0.5×)Full content description
Learning outcomesLowest (0.3×)All learning outcome text combined

Search results are ranked by relevance score. Filters (faculty, department, level, tag) apply before the search, so only the currently visible pool is searched.

Tip: Searching by module code prefix (e.g. CS1) is an efficient way to find all first-year Computer Science modules. Searching by keyword (e.g. machine learning, clinical, sustainability) searches inside content descriptions and learning outcomes.

5 Module Cards

Each module is displayed as a card in a responsive grid (3 columns on desktop, 2 on tablet, 1 on mobile). Cards are colour-tinted by faculty for quick visual orientation.

Card layout (collapsed)

  • Module code — bold monospace badge, faculty-coloured
  • Faculty abbreviation — e.g. FSE, FAH, same colour as the code; hover to see the full faculty name
  • Level badge — Undergraduate / Postgraduate, top right
  • Save icon — heart icon, top right; click to save/unsave
  • Module name — full title
  • Department · Faculty — coloured faculty dot and short faculty name
  • Intro snippet — first 160 characters of the module content description
  • Skill tags — Data and/or Digital shown only when the module is tagged; hover for a description of what the tag means
  • Skills — a purple pill shown when the module has Critical Skills mapped to it; expand the card to see the full list

Card layout (expanded — inline)

Click Content & LOs at the bottom of any card to expand it inline. The intro snippet is hidden when expanded to avoid duplication. The expanded view shows:

  • Module Content — the full content description
  • Learning Outcomes — listed as LO1, LO2, LO3… for easy reference
  • Critical Skills — where mapped, listed with their code (e.g. CS1.1), short name, and full description

Full Details modal

Click the Full details button (grid icon) at the bottom of any card to open a full-screen modal with the complete official module record. The modal shows:

  • Header badges — ECTS credits (hover for an explanation of what ECTS means), semester (1, 2, or Year-Long), International flag (sky-blue, hover for official MU definition), and UG/PG level
  • Skill tags — Data, Digital, Critical Skills, and Capped assessment flags where applicable
  • Overview — the module description as published on the official MU course catalogue
  • Learning Outcomes — numbered list
  • Critical Skills — skill codes and names, where mapped
  • Teaching & Learning — breakdown of hours: lectures, labs/practicals, tutorials, planned online learning, independent study, and total
  • Assessment — each component's type, options, weighting (%), and duration (minutes) where applicable; also shows pass standard and whether the overall result is capped
  • Supplemental / Resit — whether supplemental assessment is permitted and any conditions
  • Timetable — a link opens the official MU timetable page for the module in a new tab

The modal can be closed with the ✕ button, by pressing Escape, or by clicking outside it.

International modules

A sky-blue International badge appears on cards and in the Full Details modal for modules that are available to short-term, visiting international students at Maynooth University. Hover the badge to read the official MU definition. This flag is drawn from the official MU course catalogue.

6 Critical Skills

Critical Skills is a proposed university framework for identifying and communicating the transferable skills embedded in MU modules. Where a module has been mapped to the framework, a purple Skills pill appears on its card. Expanding the card reveals the full list of mapped skills.

The taxonomy

Ten skills are defined across four categories. Each skill has a short code (e.g. CS1.1), a short name, and a full description.

CS4.3Creativity & InnovationEthical reasoningApplying ethical principles to decision-making and innovation.

How to see Critical Skills in the catalogue

Critical Skills mappings are currently assigned to a small number of modules for demonstration purposes. The two demo modules — JGK01 and JGK02 — are a good starting point:

  1. Go to jkeatingmu.github.io/MU-finder and open the Module Catalogue.
  2. Type JGK01 in the search box. The module Data-Driven Thinking and Analysis will appear with Data and Skills pills.
  3. Click Full content & learning outcomes to expand the card.
  4. The expanded view shows four numbered learning outcomes (LO1–LO4) and four mapped Critical Skills: CS1.1, CS1.2, CS1.3, and CS4.1.
  5. Search JGK02 to see Digital Skills for Academic and Professional Practice, which carries Digital and Skills and is mapped to CS2.1, CS2.2, CS3.1, CS3.2, CS4.2, and CS4.3.
Note: These demo modules are hidden — they do not appear in the default module listing and are not included in result counts. They surface only when searched directly by code. This allows the Critical Skills feature to be demonstrated without affecting the standard catalogue view.

Status

The Critical Skills framework is a working proposal by Prof. John Keating, developed in consultation with MU Student Skills & Success. The taxonomy is subject to revision pending university approval. Skills can be assigned to any module through the module management system and will appear in the public catalogue after the next export.

7 Saving Modules

Students can save modules to a personal shortlist by clicking the heart icon (♥) on any module card. Saved modules persist in the browser's local storage — no account or login is required.

Saved filter

Once one or more modules are saved, a Saved (N) button appears near the top of the catalogue. Clicking it activates the saved filter, showing only saved modules. All other filters (faculty, department, level, search) continue to apply within the saved set.

Summary panel

When the saved filter is active, a summary panel appears above the results showing:

  • Total number of saved modules
  • How many are tagged Data and Digital
  • Which faculty abbreviations are represented
  • Which departments are represented

This gives students an at-a-glance view of the shape of their personal module shortlist — useful when planning a programme of study or exploring skill coverage.

Privacy: Saved modules are stored only in the user's own browser via localStorage. No data is sent to any server. Clearing browser data will remove the saved list. The key used is mu-module-favs, separate from the Programme Finder's saved programmes (mu-favourites).

8 Module Code Convention

MU module codes follow a consistent naming convention: a letter prefix (typically a department abbreviation) followed by a number. The first digit of the number indicates the level and, for undergraduate modules, the year of study.

First digitLevelYearExample
1UndergraduateYear 1CS101, MA101
2UndergraduateYear 2CS204, BI201
3UndergraduateYear 3CS301, PH302
4UndergraduateYear 4CS401, EN401
6Postgraduate—CS601, MA601
8Postgraduate—CS801, BI801
0, 5, 7, 9Unknown—Requires manual review

This convention was used to bulk-tag the UG_PG field and yearOfStudy field in the database for the 2,370 modules that previously had no level classification. Approximately 87 modules with edge-case first digits remain classified as Unknown and are available for review by administrators. Individual modules can be manually corrected through the admin editing interface.

What is excluded from the public catalogue

The following fields are present in the internal module management system but are intentionally excluded from the public catalogue:

  • Completed flag — an internal workflow marker indicating that a department head has finished reviewing a module; not meaningful to public users
  • Notes — free-text notes added during the tagging process, primarily used during AI/human tagging coordination; not intended for public display
  • Change log — audit trail of edits made in the management system

11 Admin: Updating the Catalogue

The public catalogue is served as a static JSON file. It does not connect to the database at runtime. To update the public catalogue after changes have been made in the module management system:

  1. Log into the MU Module Management System at http://localhost:3002 as an Administrator.
  2. Click Export for Public Finder in the button bar. A file named mu-modules-export-YYYY-MM-DD.json will download.
  3. Rename the file to modules.json.
  4. Copy it to /MU-finder/public/data/modules.json, replacing the existing file.
  5. Commit and push to GitHub: 
    git add public/data/modules.json
git commit -m "data: update module catalogue export"
git push origin main
  6. GitHub Actions will automatically build and deploy to GitHub Pages within ~2 minutes.
Frequency: Modules only change annually (new academic year). The export should be refreshed whenever significant tagging work has been completed, new tags have been introduced, or the module database has been updated for a new year.

Export format

The export JSON has the following top-level structure:

{
  "generated": "2026-03-28T...",   // ISO timestamp of export
  "totalModules": 2599,
  "faculties": [...],              // sorted faculty name list
  "index": [...],                  // per-department summary with counts
  "modules": [...]                 // full module array
}

13 Known Limitations

  • Module data is from Academic Year 2026; the catalogue will need to be re-exported for each new year
  • 282 modules are present in the university database but could not be found on the official public course catalogue (likely retired or restructured codes); their content descriptions are sourced from the original database extract only
  • Approximately 87 modules have a UG/PG classification of "Unknown" due to non-standard first digits in their module codes; these require manual review
  • 265 modules newly added from programme structure data have no Faculty assignment yet — they show no faculty colour tinting until manually assigned in the management system
  • The Data and Digital tags reflect the state at the time of export; changes made in the management system are not reflected until a new export is committed
  • The JSON file is loaded in full on first visit; on a slow connection this may take a moment. Subsequent visits are instant as the browser caches the file
  • Saved modules are stored in the local browser only — clearing browser data removes the saved list, and saves do not transfer between devices
  • The catalogue does not yet show which programmes each module belongs to; a Programme Browser showing the full programme → year → module structure is under active development
  • Additional tag types (e.g. Placement, Accredited) and admin module editing are planned for future development phases
